hexo

安装前提

安装 Hexo 相当简单。然而在安装前，您必须检查电脑中是否已安装下列应用程序：

  • Node.js
  • Git

如果没有安装，就要先安装。

正式安装Hexo**

Node和Git都安装好后,首先创建一个文件夹,如blog,用户存放hexo的配置文件,然后进入blog里安装Hexo。

执行如下命令安装Hexo：

sudo npm install -g hexo

初始化然后，执行init命令初始化hexo,命令：

hexo init

好啦，至此，全部安装工作已经完成！blog就是你的博客根目录，所有的操作都在里面进行。

生成静态页面

hexo generate（hexo g也可以）

本地启动

启动本地服务，进行文章预览调试，命令：

hexo server

浏览器输入http://localhost:4000

配置Github

首先在github新建一个仓库，仓库的名称格式必须为【your_user_name.github.io】（列如：chenkelun.github.io）

然后配置公钥: 在github上要对应自己的本地的公钥。

现在我们需要_config.yml文件，来建立关联。config.yml文件的最下方有个deploy:配置，配置如下：

deploy:

​ type: git

​ repository: git@github.com:chenkelun/chenkelun.github.io.git

​ branch: master

然后执行命令：

npm install hexo-deployer-git –save

到此就配置和关联完成了。

写作

新建一篇文章 hexo new “filed_name"然后就在目录\source_posts生成对应的filed_name.md文件

下载编辑工具，markdown(下载地址) : https://www.typora.io/

直接用markdown打对应的md文件进行编辑即可

之后只想hexo s (保存)， hexo g （部署），hexo d，就可以看到自己在github的文章

generate

生成静态文件。

选项 描述
-d, --deploy 文件生成后立即部署网站
-w, --watch 监视文件变动

该命令可以简写为

deploy

部署网站。

参数 描述
-g, --generate 部署之前预先生成静态文件
